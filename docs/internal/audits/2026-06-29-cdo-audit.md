## Backlog-ready findings

### Finding 1: Replace internal/outreach section labels on persona pages

- **Priority:** Medium
- **Status:** Agent-actionable
- **Page/path:** `/for-race-directors/`, `/for-community-races/`, `/for-marathons/`, `/for-runsignup-races/`
- **Evidence:** Live persona snapshots show public section labels such as `WHY THIS PAGE EXISTS` and `OUTREACH AUDIT PROMPTS` before useful buyer-facing content like `Common decision points before runners register.` and `What we would check first.`
- **Buyer impact:** The visitor is momentarily shown StartLine's internal page rationale/outreach framing instead of a direct buyer resource label. That can make an otherwise polished page feel like a sales-enablement artifact.
- **Recommended smallest fix:** Replace these kickers with buyer-facing labels such as `RUNNER DECISION POINTS`, `FIRST AUDIT CHECKS`, or segment-specific equivalents. Keep headings, CTAs, proof-safety caveats, routing, and page structure intact unless the copy change requires a tiny adjacent label adjustment.
- **Acceptance criteria:**
  - Persona pages no longer display `WHY THIS PAGE EXISTS` or `OUTREACH AUDIT PROMPTS` as public section labels.
  - Replacement labels are buyer-facing, race-director-friendly, and segment-appropriate.
  - Primary CTAs remain `Request a private audit`; secondary sample/checklist paths remain lower commitment.
  - No pricing, package, deposit, route, metadata, form, or proof-asset changes are included.
  - No fake proof, fake customer names, fake testimonials, or guaranteed outcome claims are introduced.
- **Verification steps:**
  - `npm run build`
  - `npm run test`
  - Source search for removed public strings.
  - Browser smoke for each persona page on the deploy preview.
  - Mobile/overflow check at 375/414/768 where practical.
  - `git diff --check`
- **Claim/proof safety notes:** This is copy-polish only. Keep generic example/proof disclaimers safe; do not replace them with implied real proof.

- **Status:** Steve-needed

- **Verification steps:**
